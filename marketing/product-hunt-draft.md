## Tagline
A privacy-first Chrome extension that scans pages for WCAG accessibility issues, pins visual annotations onto elements, and exports shareable reports — no account required.

## Description
a11y-annotator helps developers, designers, and QA catch accessibility issues faster by turning abstract WCAG failures into visible, actionable annotations.

Run 20+ automated checks directly in the browser, see color-coded pins on the offending elements, review issues in a side dashboard with WCAG Success Criterion references, then export Markdown reports or annotated screenshots you can share with teammates.

What makes it different:
- Privacy-first: all processing happens locally, no accounts, no telemetry
- No setup: install from the Chrome Web Store and scan any page
- Built for real workflows: annotations, export, keyboard shortcut

Pricing:
- Free tier: scans, visual pins, annotations, dashboard review, Markdown export, screenshot capture
- Pro: $4.99 one-time — adds CSV/JSON export, batch scan, and priority support

## Target audience
Frontend developers, UX designers, accessibility reviewers, QA engineers, and anyone shipping public-facing web apps.

## First comment / maker comment
Hey PH — I built a11y-annotator because I kept auditing the same pages over and over and wishing the failures were just... visible.

Most a11y tools I tried were either full audit platforms, heavy Lighthouse runs, or spreadsheet exports. I wanted something lighter: open a page, click once, see exactly what needs fixing, annotate it, and share it.

The result is a Chrome extension that:
- Runs 20+ automated WCAG checks
- Pins red/yellow/blue annotations directly on elements
- Lists issues with WCAG references in a side dashboard
- Exports Markdown or annotated PNG
- Works offline and never sends data anywhere

Free covers the core workflow. Pro is a $4.99 one-time upgrade if you need CSV/JSON/batch export.

Would love feedback from anyone who regularly audits UI for accessibility — what checks should I add next?
