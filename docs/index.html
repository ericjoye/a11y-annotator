Features

Everything you need for accessible websites

Automated scanning, visual annotations, and team-ready reports — all from your browser.

20+ Automated Checks

Detects missing alt text, low contrast ratios, missing form labels, empty links, heading hierarchy breaks, missing lang attributes, small touch targets, and more.

Visual Pin Overlay

Color-coded pins appear directly on problematic elements — red for errors, yellow for warnings, blue for info. See exactly where issues are at a glance.

Pin & Annotate

Click any pin to add a text note explaining the fix or assigning it to a teammate. Notes persist for the page session so your team stays aligned.

Issue Dashboard

Side panel lists all detected issues grouped by type, with severity badges and WCAG Success Criterion references like "1.1.1 Non-text Content."

Markdown Export

One-click report with page URL, scan timestamp, issue list with severity, WCAG references, element selectors, and your annotation notes.

Screenshot Capture

Download the viewport with annotation overlays as a PNG. Perfect for bug reports, design reviews, and documentation.

Privacy-First

Runs entirely in your browser. All processing is local — no data sent to external servers, no accounts needed, no tracking. Ideal for NDA-covered projects.

Keyboard Shortcut

Press Alt+A to toggle the annotation overlay on and off without reaching for the mouse.

FAQ

Frequently asked questions

Automated checks catch roughly 40-50 percent of WCAG issues. Treat this as a first-pass filter, not a full audit.

Does a11y-annotator send my data to external servers?
No. All scanning and processing happens locally in your browser. No data is transmitted to any external server, and no account is required. This makes it safe to use on pre-launch products and NDA-covered projects.
What WCAG version does it check against?
a11y-annotator checks against WCAG 2.1 Level AA success criteria. Each detected issue includes the specific WCAG reference (e.g., "1.1.1 Non-text Content") so you can look up the full guideline.
Can I use it on any website?
Yes. The extension has permissions to run on any URL. Click the extension icon on any page to scan it. It works on local development servers, staging environments, and production sites.
What's the difference between Free and Pro?
Free includes scanning, visual pins, annotations, dashboard review, Markdown export, and screenshot capture. Pro adds CSV export, JSON export, and batch scan.
Do I need to install anything else?
No. a11y-annotator is a standalone Chrome extension. Just install it from the Chrome Web Store and start scanning. It requires Chrome 88 or later with Manifest V3 support.
How do I report a bug or request a feature?
Open an issue on GitHub or email support. We read every message.
What can Pro do that Free can't?
Free covers scanning, pins, annotations, dashboard review, Markdown export, and screenshot capture. Pro adds CSV export, JSON export, and batch scan.
Does this catch every accessibility issue automatically?
No. Automated checks catch roughly 40-50 percent of WCAG issues. Think of it as a first-pass filter that saves you from repeating the same manual checks. Manual review is still required for full coverage.
