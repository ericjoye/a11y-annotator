WCAG 2.2 AA Checklist: What Every Web Audit Needs

July 13, 2026 · 8 min read

WCAG 2.2 added 9 new success criteria to the web accessibility standard, making AA compliance more comprehensive — and more work for auditors. Whether you're auditing your own site or a client's, this checklist covers every criterion you need to test for AA conformance.

What Changed in WCAG 2.2

WCAG 2.2, published in October 2023, introduced several new criteria focused on user experience for people with cognitive and motor disabilities. Key additions include:

  • Focus Appearance (2.4.11-2.4.12) — Minimum and enhanced focus indicator size and contrast requirements
  • Dragging Movements (2.5.7) — Functionality that requires dragging must have a pointer-based alternative
  • Target Size Minimum (2.5.8) — Pointer targets must be at least 24x24 CSS pixels
  • Accessible Authentication (3.3.7-3.3.8) — Cognitive function tests (like CAPTCHAs, password memory) must have alternatives

These additions mean that an audit targeting WCAG 2.1 AA is no longer sufficient for new projects that need current-standard compliance.

Building an Audit Workflow

An efficient audit process combines automated scanning with targeted manual testing. Here's a workflow that works:

  1. Run an automated scanner first — catch contrast, ARIA, alt text, and heading issues immediately. Fix these before proceeding to manual checks.
  2. Test keyboard navigation — tab through every interactive element. Focus must be visible, logical, and not trapped in modals.
  3. Test with a screen reader — NVDA (Windows) or VoiceOver (Mac). Listen to how content flows and whether landmarks are announced correctly.
  4. Check reflow and zoom — 200% zoom and 400% zoom at 1280px viewport. Content should not overflow horizontally.
  5. Review forms and errors — Submit with invalid data, verify error messages are clear and correctly associated with their fields.

How Much Can You Automate?

Based on the table above, roughly 12–14 of the ~38 AA success criteria can be partially or fully automated — about 30–35%. The rest require human judgment, screen reader testing, or manual interaction verification.

But automated checks are still the fastest way to start an audit. Fixing the automated failures first means your manual testing time is spent on the problems that actually need human expertise, not on obvious issues like missing alt text or contrast failures.
